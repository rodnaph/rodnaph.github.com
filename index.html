Sockso

Sockso is a Java project that I started many years ago with the goal of making it as easy as possible to stream and share music between myself and a non-technical friend. The solution needed to run anywhere with as few dependencies as possible, be super easy to configure, and basically just work. And that's Sockso! As long as you have Java, you just need to double click and tell it where your music is.

The project has a bunch of useful features and implementations like its own HTTP request/response code, DI via Guice, easily extendable through heavy use of patterns and interfaces, high testing coverage, and a great community of users and contributors.

 
                        class Sockso {

                            public static void main( String args[] ) {

                                Server server = new HttpServer();
                                Music collection = new MusicCollection( "/Users/rod/Music" );

                                server.share( collection );

                            }

                        }

Sockso iOS

This is a spin-off from the main Sockso project, and is an iPhone app you can use to connect to your Sockso server from your iOS device. It also integrates with Sockso's community feature so that you can browse and listen to all the community servers that people have made available too. 

                            @implementation SocksoClient

                            @synthesize iphone, ipad;

                            + (void) connectTo:(SocksoServer *)server {
                                [server showBrowsePage];
                            }

Topix

This is the top level of a few Clojure projects, and is a toy tool for doing naive bayesian classification for text on an arbitrary set of topics/categories.

Bayes

My first attempt at doing text classification was purely from my head. It worked a bit, but not particularly well. So after watching a great online class on maching learning I rewrote the classification part of Topix using a naive bayesian algorithm, and pulled it out to its own little library incase I (or anyone else) found it useful in other projects.

 
                            (ns topix.example
                              (:require [bayes.core :as bayes]))

                            (def scores (->> (load-data)
                                             (map (partial bayes/probability "sports"))))

                            (doseq [score scores]
                              (println "Score: " score))

GeTweets

While trying to find training data for Topix I started using the Twitter API to pull down tweets, but suddenly hit the 200 per page limit. So I write GeTweets to provide a Clojure sequence abstraction to pulling down a users tweets.

 
                        (ns myproject
                          (:require [getweets.core :as getweets]))

                        (doseq [tweet (getweets/fetch "rodnaph")]
                          (let [{:keys [message]} tweet]
                            (println "Message: " message)))

Binder

After reading a great article about ability based injection I decided to give it a try on a NodeJS project I had started. The concept involves moving away from the noun based approach of standard object orientated dependency injection (DI) and instead of passing the object itself around just injecting the actual ability of that object that is required. The implementation will be largely language dependent based on their features, and this one is in Javascript.

The outcome was good! It allowed very loose coupling of objects, and made it simple to move functionality around when refactoring.

 
                        function Engine() {}
                        Engine.prototype = {
                            can: [ 'goForwards', 'goBackwards' ],
                            goForwards: function() {
                                // ...
                            },
                            goBackwards: function() {
                                // ...
                            }
                        };

                        function Car() {}
                        Car.prototype = {
                            needs: [ 'goForwards', 'goBackwards' ],
                            can: [ 'driveToWork' ],
                            driveToWork: function() {
                                // ...
                            }
                        };

PTask

PHP is a single threaded language with no built-in thread/process implementation at all. There is a pcntl extension which does allow forking on OS's that support it (ie. not windows).

I'd been playing with the awesome ZeroMQ, and looking to build something with it decided on a parallel processing library for PHP. PTask allows you to trivially implement a server that can be configured to run arbitrary jobs in parallel processes. It's probably not a great idea to do this with PHP, but just a proof of concept really!

 
                        <?php
                        include 'PTask/Server.php';

                        $server = new PTask\Server();
                        $server->addJob( new ParallelJob() );
                        $server->start();
